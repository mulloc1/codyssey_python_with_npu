# CPU, GPU, NPU 차이와 딥러닝에서 NPU를 쓰는 이유

이 문서는 딥러닝 맥락에서 `CPU vs NPU`, `GPU vs NPU`를 중심으로  
왜 NPU를 별도로 사용하는지 이해하기 쉽게 정리한 복습 자료다.

---

## 1) 한 줄 정의

- **CPU**: 범용 작업을 빠르게 처리하는 "만능 지휘자"  
- **GPU**: 대량 병렬 연산을 높은 처리량으로 수행하는 "병렬 엔진"  
- **NPU**: 딥러닝 연산(MAC/행렬/텐서)을 저전력으로 가속하는 "AI 전용 엔진"

---

## 2) 아키텍처 관점의 핵심 차이

### CPU

- 소수의 강력한 코어로 순차 제어, 분기, OS 작업, 다양한 프로그램 실행에 강하다.
- 캐시/분기 예측/복잡한 제어 흐름 최적화가 잘 되어 있다.
- 장점: 유연성 최고, 어떤 코드든 실행 가능
- 한계: 대규모 텐서 연산에서 전력 대비 처리량이 낮아지기 쉽다.

### GPU

- 많은 코어(또는 SIMD/SIMT 유사 구조)로 같은 연산을 대량 데이터에 병렬 적용한다.
- 행렬 곱, 컨볼루션 같은 고정 패턴 연산에서 매우 높은 처리량을 낸다.
- 장점: 학습/추론 모두에서 높은 성능, 생태계(CUDA 등)가 성숙
- 한계: 전력 사용량과 발열이 큰 편, 지연시간/배터리 제약 환경에는 불리할 수 있다.

### NPU

- 신경망 추론의 반복 연산에 맞춘 데이터 경로와 연산 블록(MAC 어레이, 정수 양자화 친화 경로 등)을 사용한다.
- 모델을 작은 정밀도(INT8, INT4 등)로 효율적으로 실행하도록 설계되는 경우가 많다.
- 장점: **전력 대비 성능(Performance per Watt)** 우수, 온디바이스 실시간 처리에 유리
- 한계: 지원 연산/프레임워크/드라이버 제약이 GPU보다 엄격할 수 있다.

---

## 3) 딥러닝에서 중요한 지표로 비교


| 항목                | CPU   | GPU   | NPU          |
| ----------------- | ----- | ----- | ------------ |
| 범용성               | 매우 높음 | 중간    | 낮음(전용성 높음)   |
| 대규모 병렬 연산         | 낮음~중간 | 매우 높음 | 높음(추론 특화)    |
| 전력 효율             | 중간    | 낮음~중간 | 매우 높음        |
| 개발 생태계 성숙도        | 매우 높음 | 매우 높음 | 장치별 편차 큼     |
| 학습(Training) 적합성  | 낮음    | 매우 높음 | 낮음~중간(제한적)   |
| 추론(Inference) 적합성 | 중간    | 매우 높음 | 매우 높음(특히 엣지) |
| 지연시간/배터리 친화성      | 중간    | 중간    | 매우 높음        |


---

## 4) CPU vs NPU: 왜 NPU를 쓰는가

딥러닝 추론에서 CPU도 동작은 가능하지만, 아래 이유로 NPU가 필요해진다.

1. **전력 효율 차이**
  - 같은 추론 작업을 더 적은 전력으로 처리하기 쉽다.
  - 모바일/노트북/임베디드에서는 배터리 시간과 발열이 제품 품질로 직결된다.
2. **지속 성능(서멀 스로틀링 대응)**
  - CPU는 장시간 AI 연산 시 클럭 저하가 빠르게 나타날 수 있다.
  - NPU는 해당 패턴에 맞춰 설계되어 장시간 안정 동작에 유리하다.
3. **실시간 사용자 경험**
  - 음성 인식, 카메라 보정, 번역, 개인화 추천 등에서 지연시간을 줄이기 쉽다.
  - 온디바이스 처리로 네트워크 왕복 없이 즉시 반응이 가능하다.
4. **CPU 자원 해방**
  - AI 연산을 NPU로 오프로딩하면 CPU는 앱 로직/시스템 작업에 집중할 수 있다.
  - 전체 시스템 체감 성능이 좋아진다.

정리하면, `CPU vs NPU`는 "동작 가능 여부"보다  
**같은 사용자 경험을 더 낮은 전력과 안정적인 성능으로 제공할 수 있느냐**의 문제다.

---

## 5) GPU vs NPU: GPU가 있는데 왜 NPU가 필요한가

GPU는 매우 강력하지만, 모든 환경에서 최선은 아니다.

1. **전력당 성능 최적점이 다름**
  - 데이터센터/학습 환경: GPU가 최강인 경우가 많다.
  - 엣지/온디바이스 추론: NPU가 전력당 성능에서 더 유리한 경우가 많다.
2. **항상 켜져 있는 AI 기능**
  - 배경 소음 제거, wake word 탐지, 카메라 후처리처럼 상시 동작 기능은 전력 민감하다.
  - NPU는 이러한 저전력 상시 처리 시나리오에 특히 적합하다.
3. **발열/소음/배터리 제약**
  - 노트북/모바일 기기에서는 GPU를 오래 점유하면 발열/팬 소음/배터리 소모가 커진다.
  - NPU 오프로딩은 제품 체감 품질(온도, 소음, 사용시간) 개선에 직접적이다.
4. **개인정보/오프라인 처리**
  - 온디바이스 NPU 추론은 민감 데이터의 외부 전송을 줄이는 데 도움이 된다.
  - 네트워크 없는 환경에서도 일관된 AI 기능 제공이 가능하다.

정리하면, `GPU vs NPU`는 "절대 최고 성능" 경쟁이라기보다  
**목표 환경에서의 전력, 지연시간, 발열, 프라이버시를 포함한 시스템 최적화** 관점이다.

---

## 6) 딥러닝 파이프라인에서 역할 분담(실무 관점)

- **학습(Training)**: 주로 GPU(또는 대규모 가속기), CPU는 데이터 로딩/전처리 보조
- **서버 추론**: GPU/NPU/전용 추론칩 혼합, 트래픽 패턴과 비용에 따라 선택
- **엣지 추론(폰/PC/IoT)**: NPU 우선, CPU/GPU는 폴백 또는 하이브리드

실무에서는 "하나만 정답"이 아니라 다음 질문으로 결정한다.

- 목표 FPS/지연시간은?
- 배터리/전력 예산은?
- 모델 정밀도(INT8/FP16 등) 전환 가능한가?
- 프레임워크/런타임이 해당 NPU를 지원하는가?
- 실패 시 CPU/GPU 폴백 전략이 있는가?

---

## 7) NPU 도입 시 주의할 점

1. **호환성**
  - 모든 모델 연산자가 NPU에서 동작하지 않을 수 있다.
  - 일부 연산은 CPU/GPU로 폴백되어 기대 성능이 깨질 수 있다.
2. **양자화 품질**
  - INT8/INT4 최적화 과정에서 정확도 하락이 발생할 수 있다.
  - Calibration/검증 파이프라인이 필요하다.
3. **도구 체인 의존성**
  - 칩 벤더 SDK/드라이버/컴파일러 버전에 따라 성능 편차가 크다.
  - 배포 환경별 재현성 관리가 중요하다.

---

## 8) 결론

- CPU는 범용성, GPU는 고성능 병렬 처리, NPU는 저전력 AI 추론에 강점이 있다.
- 딥러닝에서 NPU를 쓰는 핵심 이유는 **전력 효율, 지연시간, 발열, 온디바이스 경험**이다.
- 따라서 "NPU가 GPU를 완전히 대체"한다기보다,  
**딥러닝 워크로드를 환경에 맞게 분업하는 것이 정답**에 가깝다.

---

## 9) 이 과제(`python_with_npu`)와 연결해서 보기

- 과제의 MAC 구현은 NPU가 잘하는 연산 패턴의 축약판을 직접 체험하는 과정이다.
- `CPU로도 구현은 가능`하지만, 실제 제품에서는 같은 계산을 **NPU로 오프로딩**해  
전력과 응답성을 개선한다는 점이 핵심 학습 포인트다.
